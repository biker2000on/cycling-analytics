### E2E Test Implementation

**File**: `frontend/e2e/upload-and-calendar.spec.ts`

**Test Flow**:

1. **Login**:
   - Reads credentials from `.env.test` (never hardcoded)
   - Parses `LOCAL_USER` and `LOCAL_PASSWORD`
   - Logs in via `/login` page
   - Waits for redirect to `/activities`

2. **Upload .zip files**:
   - Uploads all 4 .zip files from `.fitfiles/` directory:
     - `21796666064.zip`
     - `21817961064.zip` (contains nested .zip)
     - `21817965502.zip`
     - `21829011561.zip`
   - Uses absolute paths: `C:/Users/biker/projects/cycling-analytics/.fitfiles/`
   - Waits for upload summary message

3. **Wait for processing**:
   - Monitors upload status indicators
   - Waits for all "Processing" states to complete (max 90s)
   - Verifies "Done" statuses appear

4. **Verify activities**:
   - Waits for `onUploadComplete` callback to refresh list
   - Verifies activity table is visible
   - Confirms at least some rows exist

5. **Navigate to calendar**:
   - Clicks calendar nav link
   - Waits for page load

6. **Test infinite scroll**:
   - **Scroll up**: Triggers loading older months, verifies month count increases
   - **Scroll down**: Triggers loading newer months (up to limit)
   - **Today button**: Clicks and verifies current month is visible

7. **Verify calendar data**:
   - Looks for days with `.day-cell-has-activity` class
   - Confirms at least 1 day has activities from uploaded files
